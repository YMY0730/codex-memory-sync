# 🧠 Codex Memory Sync

> *你的 AI 编程助手，不应该每台设备都是"新来的实习生"。*

---

## 一句话说清楚

Codex Memory Sync 让你的 Codex 和 OpenCode **记住你**。你在任何一台电脑上积累的项目理解、对话上下文、工作偏好，都能无缝迁移到另一台设备、甚至另一个 AI 工具。**一次配置，终身跟随。**

---

## 它解决了什么

AI 编程助手最大的浪费不是模型调用，而是**重复教育**。你花了三周在一个项目上，让 AI 理解了架构、记住了偏好、积累了上下文。换台电脑——一切归零。换到 OpenCode 试试——从零开始。

这就是 Codex Memory Sync 要消灭的问题：

- **跨设备** → 公司台式机、家里笔记本、远程服务器，AI 认知状态完全同步
- **跨工具** → Codex ↔ OpenCode，积累的上下文自由迁移，不被任何平台绑定
- **离线备份** → 导出加密包，U 盘拷贝即用，不依赖云服务也不怕断网

---

## 核心卖点

### 🧠 记忆跟随

你在 Codex 里让 AI 记住的项目规范、命名偏好、架构决策——这些不是"临时指令"，是你的**数字认知资产**。Codex Memory Sync 把它们打包、加密、同步，像 iCloud 同步照片一样自动流转。

### 💬 对话不丢失

每一段复杂的调试对话、每一个深思熟虑的架构讨论，都是你花了时间和 Token 换来的。它们不应该像聊天记录一样被遗忘。支持 Codex `.jsonl` 和 OpenCode `SQLite` 两种格式的完整会话提取——**连思考过程都能保留**。

### 🔄 打破工具围墙

Codex 用腻了想试试 OpenCode？或者反过来？不用从零开始教新工具。Codex Memory Sync 是业内首个支持 **两大 AI 编程助手双向记忆移植** 的工具——AGENTS.md、Skills、会话记录、Todo 任务，全都能一键迁移。

### 📂 按项目组织，按目录清晰

OpenCode 的数据不再是混在一起的"global"杂烩。每个工作目录独立呈现——`D:\my-project` 有 8 个会话、`D:\client-work` 有 3 个会话——和你大脑里的项目概念完全一致。

### 🔒 安全第一

AES-256-GCM 军用级加密。导出包可设密码保护。导入 OpenCode SQLite 前自动备份，万无一失。本地模式无需任何云服务。

## 和同类的对比

| | 手动复制文件夹 | Codex Memory Sync |
|---|---|---|
| **操作方式** | 找到 `.codex` 目录，手动拷贝 | 一键导出，双击导入 |
| **跨工具** | 完全不支持 | Codex ↔ OpenCode 双向 |
| **加密保护** | 无 | AES-256-GCM |
| **会话预览** | 看不到内容 | 聊天气泡时间线 |
| **按项目组织** | 做不到 | 按工作目录清晰分组 |
| **自动同步** | 无 | 后台守护进程 |

## 立即开始

```bash
cd codex-memory-sync
pip install -e .
python web_server.py
```

浏览器打开 `http://127.0.0.1:8899`，三分钟完成第一次跨设备迁移。

> 💡 **推荐使用网页端** — 功能最全、无需安装桌面环境、暗色主题舒适护眼。也可以用 `python cli.py gui` 启动桌面版。
